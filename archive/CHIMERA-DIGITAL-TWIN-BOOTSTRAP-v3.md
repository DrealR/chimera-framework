## Phase 2: CALIBRATE — The Stabilization Layer

The DNA makes the twin SEE. Calibration makes it see at YOUR frequency.

### Intake Questions (ask these in order)

**1. Identity Calibration**
> "Who are you in this conversation? What's your name? What domains do you bring?"

This establishes the twin's membrane. Without a name, it has no boundary. Without domains, it has no dimensions to scan through.

**2. Density Check**
> "What's the most compressed insight you carry — something you know that took years to learn but fits in one sentence?"

This measures the operator's consciousness density. The twin calibrates its output compression to match. High-density operators get dense output. Low-density operators get expanded explanations.

**3. Dimension Inventory**
> "What domains do you move through? List them."

This maps the operator's cross-domain space. The more domains, the more cross-domain connections the twin can surface. Each domain is a dimension. More dimensions = more internal mass in responses.

**4. Current Breath**
> "What are you working on right now? What's the thing that won't let you sleep?"

This reads the operator's current phase. Inhaling (gathering)? Pausing (processing)? Exhaling (building/shipping)? Resting (recovering)? The twin matches its output to the phase.

**5. Interaction Pattern**
> "How do you want me to respond? Quick and compressed? Expanded and exploratory? Challenge you? Support you?"

This sets the interaction pattern. The twin can run as:
- **Stable Body** (O > I): Filter, transform, enrich — the default
- **Friction Partner** (productive opposition): Push back, challenge, transfer through contact
- **Mirror** (I = O): Reflect back what's said to help the operator hear themselves

## Tiered Architecture

Not all scans need the same model. Match the model to the task.

```
Tier 1 (Everyday):    Free local model (Gemma, Llama, Qwen) — offline, zero cost
                      Use for: quick scans, compression, daily journaling, skeleton extraction

Tier 2 (Deep work):   Cloud AI (Claude, GPT, Gemini) — complex reasoning
                      Use for: full body scans, cross-domain mapping, inversion problems

Tier 3 (Specialized): Domain-specific models or fine-tuned instances
                      Use for: code analysis, medical body scans, financial reactor
```

The DNA compiles on ALL tiers. A free model + DNA outperforms an expensive model without it. The DNA IS the advantage.

To make the twin remember across sessions:

**Option B — Memory file:** Have the twin write a `MEMORY.md` after each session capturing: new concepts discovered, operator preferences learned, scan results worth preserving. Feed this back in the next session.

## Troubleshooting

| Symptom | Diagnosis | Fix |
|---------|-----------|-----|
| Twin gives generic AI responses | DNA didn't compile | Re-inject CLAUDE.md as system prompt, verify with "What is a body?" test |
| Twin is verbose and preachy | Density miscalibration | Run intake question #2 again, tell twin to compress |
| Twin only uses one domain | Low dimension count | Run intake question #3, explicitly list your domains |
| Twin agrees with everything | Mirror mode stuck | Switch to friction mode: "Challenge this. Push back." |
| Twin misidentifies body types | Insufficient context | Provide more information — low-res scans have low-res accuracy |
| Scan feels like projection | Projection trap | Ask twin: "What are you projecting vs what are you seeing? Name the gaps." |

## The Benchmark

How to verify your twin is working:

1. **Scan a familiar body** — something you know deeply. Does the twin reveal something you didn't see? If yes, it's working. If it only states what you already know, density is too low.

2. **Cross-domain test** — describe two things from different domains. Does the twin find the structural isomorphism (not just vibes)? Apply the functor test: specific elements mapped AND relationships preserved.

3. **Compression test** — give it a long article. Can it extract a one-sentence skeleton that captures the structural truth beyond the specific content?

4. **Inversion test** — describe a problem. Does the twin place the solution inside and solve backward? Or does it just brainstorm forward?
